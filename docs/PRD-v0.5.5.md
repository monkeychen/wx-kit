# wx-kit v0.5.5 产品需求文档（迭代 PRD）

> 本文件是 **v0.5.5 迭代**的需求源头与验收依据。两个主题:文库长列表导航(安哥 2026-07-17 提出「滚动好久才看到下面的文章」)+ 按公众号下载的关键词过滤(GitHub issue #1)。
> 实现计划见 `docs/plans/2026-07-17-m23-m24-library-nav-and-keyword-filter.md`;状态/进度见 `ROADMAP.md`。

## 1. 一句话定义

把文库首屏从「一面要滚很久的卡片长墙」变成「一屏尽览的公众号目录」;把按公众号批量下载从「只能按数量/日期」升级为「可再按标题关键词筛选(仅含/排除)」。

## 2. 背景

- **文库滚动痛点**:115 篇文章、分组默认全展开、折叠状态不持久 → 卡片模式要滚很久才到下面的文章。「加载更多/分页」不对症(本地数据无等待理由,分页只是把滚动换成点击);对症的是**目录化 + 导航锚点**。
- **issue #1(2026-06-29,ghzjsny)**:按公众号下载时无法限定关键词,希望「仅下载含关键词」或「排除含关键词」的文章。可行性已评估:列表阶段即有标题,列出→下载之间加标题过滤,零额外请求;过滤基于**标题**(正文需下载后才有),且在指定抓取范围内筛(不为凑数翻页,避免加重频控)。

## 3. 功能需求

### R1 · 分组默认折叠 + 折叠记忆(里程碑 M23)

- 分组视图(卡片/列表)各组**默认收起**,组头显示「公众号 · N 篇」——首屏即目录。
- 展开状态持久化(`AppSettings.libraryExpandedGroups: string[]`,存展开集):下次进文库保持上次布局;新出现的公众号默认收起;从未设置过(首次升级)= 全收起。
- 工具栏「全部展开 / 全部收起」一键开关(按当前状态切换文案)。
- 不分组的平铺模式行为不变。

### R2 · 粘性组头(里程碑 M23)

- 滚动时当前组头钉在视口顶部(卡片视图 `.ghead`、列表视图 `.lgrp-head`;列表视图列头 `.lhead` 一并钉住,组头钉在列头之下)。
- 随时可点粘住的组头收起当前组,直接落到下一组。

### R3 · 回到顶部(里程碑 M23)

- 文库页悬浮「回到顶部」按钮(antd FloatButton.BackTop),滚动超过一屏后出现。

### R4 · 按公众号下载的关键词过滤(里程碑 M24,issue #1)

- **语义**:在「数量/日期范围列出的文章」内按**标题**过滤后下载。`include`(仅含:标题含任一关键词才下载)与 `exclude`(排除:标题含任一关键词即跳过)可单用可并用(先 include 后 exclude,exclude 优先);关键词多个逗号分隔,匹配不区分大小写。
- **core**:`crawlAccount` 支持注入关键词过滤,列表后、`onListed`/下载前应用;`CrawlSummary` 增 `filteredOut`(被过滤篇数,可选字段)供透明呈现。
- **CLI**:`crawl <名> --include <kw,kw>` / `--exclude <kw,kw>`,JSON 输出含 `filteredOut`。
- **GUI**:「下载 · 按公众号」范围行下加一行可选的关键词筛选——**操作类型下拉(仅下载含 / 排除含)+ 单个关键词输入**,留空即不过滤;结果反馈体现「列出 N 篇,按关键词过滤后下载 M 篇」。
  > 设计修订(2026-07-18 安哥):初版做成「仅含」「排除」两个并排输入框,两者并填时的组合语义用户无从知晓——互斥应由**结构**保证,改为下拉选操作类型 + 单输入。core/CLI 的 include+exclude 组合语义保留(agent 场景有文档约定)。
- 发版后在 issue #1 回复说明并关闭。

**存储影响**:`AppSettings` 新增 `libraryExpandedGroups`(可选,向后兼容);下载历史**不记录**关键词(「再来一次」不回填关键词,见非目标)。

## 6. 非目标

- **「加载更多」/分页/虚拟滚动**——分页不对症;千篇量级实测(2026-07-18)后加 `content-visibility: auto` 即恢复流畅(23→52fps),虚拟滚动推迟到万篇级再议(ROADMAP 候选有实测数据)。
- **正文关键词过滤**——正文需下载后才有,与「下载前筛选」目标矛盾;标题过滤是诚实边界。
- **为凑满数量翻页补抓**——「最近 N 篇里筛」而非「筛出 N 篇」,不加重频控。
- **下载历史记录关键词/「再来一次」回填关键词**——历史 source 结构不动,需要时单议。
