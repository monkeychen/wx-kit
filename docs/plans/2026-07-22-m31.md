## 步骤(bite-sized,TDD,每步跑 `npm test`)

### R5 · 修 mac CLI 程序坞冒图标(最先修,一行)

- **S1** `electron/main.ts` CLI 分支(`isCliInvocation` 判定后、`app.whenReady()` **前**)加 `if (app.dock) app.dock.hide()`。
  - `app.dock` 只 mac 存在;放 whenReady 前是为压在 dock 图标完全显示前。
  - GUI 分支不动。PDF 离屏 BrowserWindow 不受 dock.hide 影响。

### R4 · `-h` 加仓库 URL(一行)

- **S2** `src/cli/index.ts` 的 `addHelpText('after')` 末尾(`各命令详情:wx-kit help <命令>` 之后)追加:`\n仓库:https://github.com/monkeychen/wx-kit(可读 README.md / issues / releases 深入了解)`。
  - 硬编码常量(仓库地址永不变)。

### R3 · library CLI 排序(默认 publishTime desc)

- **S3(红)** 新建 `tests/core/library-sort.test.ts`,断言 `sortArticles`:
  - `publish` desc:publishTime 大的在前;`asc` 反之。
  - `download` / `title`(`localeCompare('zh')`)方向正确。
  - **空 publishTime 恒置末尾**(升降序皆然)。
  - 不改输入(返回新数组)。
- **S4(绿)** 新建 `src/core/library-sort.ts`,把 `sortArticles` + `SortKey` + `SortDir` 从 `src/renderer/library-view.ts` 搬来(逻辑原样);`library-view.ts` 改 `export { sortArticles, type SortKey, type SortDir } from '../../core/library-sort'`(re-export 保持渲染层 import 兼容、GUI 行为零变化)。
- **S5** `src/cli/index.ts` 的 `library list` / `library search`:
  - 加 `.option('--sort <publish|download|title>', '排序字段', 'publish')` 与 `.option('--order <asc|desc>', '升降序', 'desc')`。
  - `outJson` 前对 `items` 跑 `sortArticles(items, opts.sort, opts.order)`。
  - 默认即 `publish desc`(agent 直接 `library list` 就是最近在前)。

### R1 · 订阅部分检查(行内按钮 + CLI --accounts)

- **S6(红)** `tests/electron/subscription-check.test.ts`(若无则新建)增 `fakeids` 过滤断言:给 `runSubscriptionCheck` 传 `fakeids: ['a']`,只查 a、不动其他号的水位;空选中集 → `note: 'no-accounts'`。
- **S7(绿)** `electron/services/subscription-check.ts` 的 `runSubscriptionCheck(trigger, deps)` 增可选 `fakeids?: string[]`:`accounts = (await subs.list()).filter(a => a.subscribed)` 之后追加 `fakeids ? accounts.filter(a => fakeids.includes(a.fakeid)) : accounts`。日志 `accounts: accounts.length` 天然对(本次实际查的号数)。
- **S8** `electron/ipc.ts` 的 `subscriptions:checkNow` 增可选 `fakeids?: string[]` 入参透传;`runSubscriptionCheck('manual', { ..., fakeids })`(runSubscriptionCheck 的闭包包装要相应加形参)。`preload.ts` + `src/renderer/api.ts` 的 `subscriptionsCheckNow(fakeids?: string[])` 同步。
- **S9** `src/cli/index.ts` 的 `subscription check-now` 增 `--accounts <csv>`:`runCli` 里把它拆数组,经 ipc? ——CLI 不走 ipc,直接调 `runSubscriptionCheck('manual', deps)` 传 `fakeids`。skill 待收尾刷。
- **S10** `src/renderer/pages/Subscriptions.tsx`:
  - 顶部「检查更新」按钮文案 → 「检查全部」。
  - 每行 actions 末尾加「检查」(`<a data-testid="subs-check-one" onClick={() => checkOne(a)}>`)。`checkOne(a)` = `setChecking(true)` + `api.subscriptionsCheckNow([a.fakeid])` + `load()` + finally 解禁。行内按钮与顶部共享 `checking` 状态(in-flight 守卫已在 ipc 层,UI 层用 `checking` 置灰两者;正在跑时全置灰 = 并入语义)。
- **S11(收尾刷 skill)** `agent/wx-kit-skill/`:
  - `SKILL.md` 速查表:「单独检查某号」加 `wx-kit subscription check-now --accounts <fakeid>`;「最近文章清单」加 `wx-kit library list`(默认 publish desc)。
  - `references/commands.md`:`subscription check-now` 加 `--accounts`;`library list`/`search` 加 `--sort`/`--order`。
  - `references/recipes.md`:加「每天拉所有公众号最近文章清单」(`library list` 取前 N 条)范例。
