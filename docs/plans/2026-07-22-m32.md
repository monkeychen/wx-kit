# M32 实现计划 — 文库同步到个人站点(v0.8.0 R2)

## 已核实的事实(实现按这些来,不再回头查)

- **wx-kit 产物**:`<库根>/<公众号>/<日期_标题>/` 下有 `content.md`(frontmatter 含 `title/account/author/publishTime/source/downloadTime`,紧跟一个与 title 同文的 `# H1`)、`meta.json`(字段见上)、`images/img-N.<ext>`;正文图片引用形如 `![](images/img-1.png)`。
- **site 规范**:`content/posts/YYYY-MM-DD-<slug>/index.md` + 图片**同目录**相对引用(`./img-1.png`);frontmatter strict,微信导入只写 `title/date/source: wechat`;slug `^[a-z0-9-]+$` 且不以连字符开头/结尾、全局唯一;目录日期 = frontmatter date。
- **原子写入**:现有 `atomic-write.ts` 只做「单文件 tmp→rename」,**不覆盖目录**。site 的做法是 mkdtemp 暂存目录 → 写全 → `rename` 整个目录。M32 需要目录级原子,**不复用 atomic-write,在 site-sync 内实现**(与 site `commitPostImport` 同构)。
- **settings**:`AppSettings` + `defaults()` 两处加字段即可(`get()` 用 `{...defaults, ...raw}` 合并,老配置文件自动补默认值,无需迁移)。

## 步骤(TDD,每步跑 `npm test`)

### 核心层 `src/core/site-sync.ts`(纯逻辑,可单测)

- **S1(红)** `tests/core/site-sync.test.ts` 写 `validateSlug` 断言:合法 `nanxin-tech-analysis`;非法——大写、下划线、空格、中文、以 `-` 开头/结尾、空串;批量内重复;与已有 slug 冲突。各返回具体原因文案。
- **S2(绿)** 实现 `validateSlug(slug: string, taken: Set<string>): { ok: true } | { ok: false; error: string }`。
- **S3(红)** `buildSitePost` 断言(喂真实形态的 content.md 字符串):
  - 产出 `dirName === '<YYYY-MM-DD>-<slug>'`,date 取自 `meta.publishTime` 前 10 位。
  - `indexMd` frontmatter **只含** `title/date/source: wechat`(strict schema),title 用 JSON.stringify 转义(对齐 site `buildIndexMd`)。
  - 正文**去掉 frontmatter 段**、**去掉紧随的首个 H1**(site 已有 title,避免重复)。
  - 图片引用 `](images/` → `](./`。
  - 无 publishTime → 返回错误(不静默用今天:目录日期必须等于发布日期)。
- **S4(绿)** 实现 `buildSitePost(meta, contentMd, slug): { ok: true; dirName; indexMd } | { ok: false; error }`。

### 落盘 `commitSitePost`(目录级原子,依赖注入 fs 以便测)

- **S5(红)** 断言:正常写入后 `<postsRoot>/<dirName>/index.md` + 图片齐全;**中途失败(populate 抛错)不留残留目录**;目标已存在 → 报 slug 冲突不覆盖。
- **S6(绿)** 实现:`mkdtemp(join(postsRoot, '.staging-'))` → 写 index.md + 复制 `images/*` 到暂存**同级**(不建 images 子目录)→ `rename(staging, target)`;`finally` 未提交则 `rm -rf` 暂存。
  - 暂存放 postsRoot 内(同一文件系统才能 rename;`.staging-` 前缀 + 失败即删)。
  - 目标已存在:先 `access` 判定,存在即返回 `{ ok:false, error:'slug 已存在:<dir>' }`(**不覆盖**)。

### 编排 `syncArticlesToSite`(GUI/CLI 共用)

- **S7** `syncToSite(items: Array<{ meta, dir, slug }>, postsRoot, deps)`:串行逐篇 → 读 content.md → buildSitePost → commitSitePost;**单篇失败不阻断**,汇总 `{ succeeded, failed, results }`。
  - 提交前统一预检:批量内 slug 唯一 + 与 postsRoot 已有目录(读一次 `readdir`)不冲突 + 格式合法。

### 设置
