> 目标:导出素材后,**不必离开窗口、不必记路径、不必自己拼提示词**,一步递给 agent。

## c 方案调研结论:不做「直接唤起 Claude Code」(实测后的判断,非能力所限)

- **能力上可行**:本机 `claude` 在 PATH(`~/.local/bin/claude`),CLI 签名 `claude [options] [prompt]`,
  可 `open -a Terminal <生成的 .command>` 起一个带提示词的新会话。**没有 URL scheme**(官方未提供)。
- **但 UX 上更差**,故不做:唤起的是**新终端 + 新会话 + 陌生 cwd**(通常是 home),
  而安哥的真实用法是「已经开着一个 Claude Code 会话」——他要的是**把上下文粘进当前会话**,
  不是另起一个不在项目目录里的会话。a 方案(复制指令)恰好服务这个真实路径,还与任意 agent 通用
  (Codex/Cursor/网页版都能粘)。
- 结论:**做 a+b,不做 c**。若将来 Claude Code 提供 URL scheme 或「附加到当前会话」的入口,再评估。

## 步骤(TDD)

### 核心层(纯函数,可测)

- **S1(红)** `tests/core/material-export.test.ts` 增 `buildAgentPrompt` 断言:
  - 含清单绝对路径、篇数;
  - 说明「每篇正文在 contentPath 指向的 content.md」;
  - 引导 agent **先读清单再确认选题**(不要一上来就写);
  - 路径原样嵌入(含中文/空格不破坏)。
- **S2(绿)** `src/core/material-export.ts` 导出 `buildAgentPrompt(path: string, count: number): string`。

### 主进程

- **S3** `electron/ipc.ts` 加 `clipboard:write`(用 electron `clipboard.writeText`)。
  不用渲染层 `navigator.clipboard`——file:// 下非安全上下文,行为不稳。
- **S4** `library:exportMaterial` 返回值增 `prompt` 字段(由 `buildAgentPrompt` 生成),
  让渲染层无需重复拼串;CLI `library export` 的 JSON 同步带上 `prompt`(agent 直接可用)。

### 渲染层

- **S5** `Library.tsx` 的 `exportMaterial` 成功后由 `message` 改为 **Modal**:
  - 显示篇数 + 清单路径(可选中复制);
  - 主按钮 **「复制给 agent 的指令」**(复制后 message 提示「已复制,粘进 Claude Code 即可」);
  - 次按钮「复制路径」「打开文件夹」。
  - 主按钮居首——它是这条工作流的默认动作(为目标设计,不为功能设计)。

### 文档

- **S6** `agent/wx-kit-skill/references/commands.md` 补 `library export` 输出的 `prompt` 字段。
