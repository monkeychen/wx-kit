## 技术选型:表格规则自写,不引 `turndown-plugin-gfm`(偏离 PRD 倾向,已论证)

PRD R1 倾向「引入 turndown-plugin-gfm」。**实现前实测后改为自写**,理由:

1. 插件产出的表格骨架正确,但微信每个单元格都包了 `<section>`,被 turndown 当块级元素 → 裸换行 → **非法 GFM**。要修就得覆盖插件的 `tableCell` 规则,等于插件的核心部分我们照样要自己写。
2. 剩下的部分(遍历 `tr`/`th`/`td`、补分隔行)约 40 行,边界(无 thead、管道转义、colspan)对微信这种规整表格是确定性的,自写反而可控、可测。
3. 少一个依赖 = 少一份 npm 包体积与供应链面(`dist-npm` 的 dependencies 是手工维护的白名单)。

代价:通用性不如插件(极端嵌套表格/复杂合并单元格会退化)。取舍记入 devlog。

## 步骤(TDD,每步跑 `npm test`)

### R1 · markdown 表格保真

- **S1(红)** `tests/core/export-markdown.test.ts` 增 `describe('GFM 表格')`,先写断言:
  - 标准表(带 `<thead>`)→ `| 列A | 列B |` + `| --- | --- |` + 数据行,且**全文无裸换行破行**。
  - 无 `<thead>`(纯 `<tbody>`)→ 首行 `<tr>` 当表头。
  - **微信真实形态**:单元格为 `<td><section><span leaf="">2025年业务</span></section></td>` → 输出 `| 2025年业务 |`,单行紧凑。
  - 单元格含 `|` → 转义为 `\|`;含 `<br>`/多段 → 合并为一行(空格分隔)。
  - 单元格含行内格式 `<strong>`/`<a>` → 保留 `**x**` / `[x](url)`。
  - 空表格(无行)→ 不产出残缺表格(输出空)。
- **S2(绿)** `src/core/exporter/export-markdown.ts` 加 `td.addRule('gfmTable', { filter: 'table', replacement })`:
  - 用 `getElementsByTagName('tr')` 取行;每行取 `th,td` 子元素。
  - 单元格内容:`td.turndown(cell.innerHTML)` 得到行内 markdown → **压平**:`.replace(/\n+/g, ' ').replace(/\s+/g, ' ').trim()` → 再 `.replace(/\|/g, '\\|')`。
  - 表头:有 `<thead>` 用其首行;否则用第一行 `<tr>`。列数按表头列数,数据行不足补空、超出截断。
  - 输出前后各留空行。
- **S3** 确认既有断言(代码块/图片/标题)全绿——表格规则不得溢出到非表格内容。

### R2 · 应用内版本号

- **S4** `electron/ipc.ts` 加 `ipcMain.handle('app:version', () => app.getVersion())`;`preload.ts` + `src/renderer/api.ts` 加 `appVersion(): Promise<string>`。
- **S5** `MainLayout.tsx`:`useEffect` 取版本,刊头 `brand-mark` 后加 `<span className="brand-ver">v{ver}</span>`;`index.css` 加 `.brand-ver`(字号 11px、色 `--ink-faint`、字距正常——**低于 brand-mark 一个层级**,不喧宾夺主)。
- **S6** `Settings.tsx` 末尾加「关于」`setting-block`:版本号 + 项目主页 + Releases 两个外链(经已有 `shell.openExternal` 通道;若无则复用 `api.reveal` 之外新增最小 IPC)。
- 版本一律来自 `app.getVersion()`,**不硬编码**。

### R3 · 标题栏文案去重

- **S7** `electron/main.ts` `title: 'wx-kit'` → `title: ''`;`index.html` 的 `<title>wx-kit</title>` → `<title></title>`。
  **两处必须同改**——HTML title 加载后会覆盖 BrowserWindow 的 title。
