# wx-kit v0.5.2 — 修复命令行入口在 macOS 上崩溃（2026-07-10）

## 修复

**命令行快捷方式从软链改为 wrapper 脚本（M20）**——修复 `wx-kit download` 等命令报 `Unable to find helper app` + GPU/网络服务连环崩溃的致命问题：

- 根因：此前 `~/bin/wx-kit` 是指向应用内层二进制的 **symlink**,macOS 上 Electron 按「被调用的路径」定位同 bundle 的 Helper 子进程 app,经软链调用必然找不到 → 一切需要子进程（GPU/网络/渲染,即 `download`、PDF 导出）的命令崩溃。`--version`/`--help`/`library list` 等纯主进程命令不受影响,故此前未暴露。
- 修复：快捷命令改为 shell wrapper 脚本（`exec` 应用真实路径,与 VS Code `code` 命令同款做法）,Helper 定位恢复正常。
- **旧软链自动修复**：升级到本版后**打开一次 GUI 即自动完成**——应用启动时检测到旧版软链会静默升级为 wrapper,无需任何手动操作。

## 验证

- 单测 260 全绿（wrapper 生成 / 状态矩阵含 legacy / force 升级覆盖）、tsc / lint / Playwright GUI e2e 全过。
- 真机:旧 symlink → 开 GUI → 自动替换为 wrapper（内容/可执行位断言）;经 wrapper 入口跑 `download --formats md,pdf` 正常无报错。
