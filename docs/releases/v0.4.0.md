# v0.4.0 · 2026-06-23

「文库供料 agent + 存储加固」迭代。把文库从「自己看的归档」升级为**可被 AI agent 消费的素材源**——选中文章一键导出为结构化素材，交给应用之外的 Claude Code skill 驱动「选题 → 写作」的人在环中创作；并在此之前先把文件存储地基夯实。

## 🧱 存储加固（M13）

把 `library.json` / `history.json` / `subscriptions.json` 三个索引的写入做得抗崩、抗并发：

- **原子写**：改为「写临时文件 → 原子 `rename` 替换」，进程中途被杀 / 断电不再把索引写成半截 JSON。
- **写串行化**：同一索引文件的「读-改-写」按文件加锁串行，杜绝后台订阅自动下载与前台操作并发时的丢更新。
- **损坏可重建**：新增「重建索引」能力——扫描库根各文章目录的 `meta.json` 重建 `library.json`。CLI `library rebuild` + 设置页「重建索引」按钮；索引损坏的提示也从「手删重来」改为「可重建修复」。

## 📤 供料能力（M14）

把选中的文库文章变成 agent 能直接吃的结构化素材：

- **`library export` CLI**：输出 JSON 清单（每篇含标题/公众号/作者/发布时间/原文链接/目录/`content.md` 绝对路径，**正文不内联**），选料器 `--ids` / `--since <日期>` / `--account <公众号名>` / `--all`（无选料器时报错，避免误导全库）。
- **文库 GUI「导出选中为素材」**：复用文库多选，选中后一键把素材清单写到库内 `exports/<时间戳>.json`，提示路径、可在文件夹显示。外部 skill 接这个文件即可。

## 🤖 贯通样例 skill（M15）

仓库新增 `agent/wx-kit-compose`——一个**消费 wx-kit 的 Claude Code skill** 参考实现（与应用代码隔离，只经 CLI / 导出文件取数据，绝不 import 应用代码）：

- 编排「选料 → 选题候选 → 🛑你拍板 → 委派 `khazix-writer` 出初稿 → 🛑你审定」，两个检查点强制**人在环中**。
- 选题轻量内联，写作委派给写作 skill（`khazix-writer`），wx-kit 只供料、不内置创作模块。
- 安装与用法见 `agent/README.md`。

## 边界与立场

wx-kit 守住「下载器 + 文库」本分，**只供料**；选题/写作/审阅的编排活在应用之外的 skill 里，人在环中。文库素材用于选题/研究/启发 + 人工主导创作，**不洗稿、不自动转发**。
