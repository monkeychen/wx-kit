# wx-kit v0.5.1 — 支持文字消息与图文消息（2026-07-09）

## 修复 / 新增

**非标准消息类型解析（M19）**——此前下载微信「文字消息」和「图文消息（小绿书）」会得到坏数据，现已完整支持：

- **文字消息**（纯文字短内容，`item_show_type: '10'`）：
  - 修复标题被塞入整篇正文（带字面 `\n`）导致的超长标题、乱码目录名、阅读器标题爆版。
  - 修复正文一片空白——正文实际藏在页面脚本变量里（前端渲染），现从 `text_page_info.content` 提取并分段。
  - 标题取正文首行截断（≤30 字 + `…`），与微信客户端列表展示一致。
- **图文消息 / 小绿书**（`item_show_type: '8'`）：
  - 修复正文空白与图片全丢：文字从 `cgiDataNew.content_noencode` 提取，图片从 `window.picture_page_info_list` 提取主图（自动排除水印图与分享封面），全部走既有图片本地化管线。
- **通用加固**：og:title / og:description 兜底值中的字面转义序列（`\n`、`\x0a` 等）统一清洗，任何路径都不再渗入标题/摘要。

md / html / pdf 导出、GUI 阅读器、CLI 全链路自动受益（解析层单点根治，下游零改动）。

## 说明

- 已下载的坏记录不做自动迁移：在文库删除后重新下载即可。
- 图文消息中的实况图（live photo）只保存静态图，不下载视频。

## 验证

- 单测 255 全绿（含两种类型的真实页面裁剪 fixture）、tsc / lint / Playwright GUI e2e 全过。
- 两条真实 URL 全格式（md/html/pdf/meta）下载 + GUI 阅读器真机验证（文字消息 12 段正文、图文消息 7 图 `naturalWidth>0`）。
