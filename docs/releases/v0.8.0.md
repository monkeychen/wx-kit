# wx-kit v0.8.0 — 让内容流到该去的地方(2026-07-22)

本版两条主线:**让 agent 用 CLI 用得更顺**(订阅可按号点检、列表默认按发布时间排好序、`-h` 给出仓库地址),以及**让下载到的文章能一键流进个人站点**。外加修掉一个 mac 上的老毛病。

## 新增

**文库/CLI 同步到个人站点(M32 · R2)**——把已下载的文章按 Astro 站点的发布规范直接生成到 `content/posts/`:

- 产出 `YYYY-MM-DD-<slug>/index.md`,目录日期取自文章的 **publishTime**;frontmatter 只写 `title / date / source: wechat`(严格对齐站点 strict schema,多一个字段站点就构建失败)。
- 正文自动去掉 wx-kit 的 frontmatter 与重复的首个 H1;**图片摊平到同目录**(`images/img-1.png` → `./img-1.png`),站点侧零手工调整。
- **目录级原子写入**:先写暂存目录、写全再整体 `rename`,中途失败不留半成品;**slug 与已有目录冲突时不覆盖**,该篇记失败、不阻断其他篇。
- 这是**纯个人需求**,故默认隐藏:设置页新增「站点同步」开关(默认关)+ 目标目录;**开关只管 GUI 按钮可见性,CLI 通道不受限**(agent 配好目录即可用)。
- GUI:文库选中文章 → 底部指令区「同步到站点」→ 逐篇填 slug → 结果汇总(成功列目录、失败列原因)。
- CLI:`wx-kit site sync --ids <id> --slug <slug>`,批量用 `--slugs <id>=<slug>,...` 或 `--slugs-file`(**按 id 映射,不靠位置对应**)。

**订阅支持按号点检(M31 · R1)**——以前只有一个全量「检查更新」,想单独看某个号有没有更新没地方点:

- 订阅页每行新增「检查」动作,顶部按钮改名「检查全部」(语义对照)。
- **频控纪律不变**:子集照样串行 + 账号间随机延迟;与全量检查共享同一个 in-flight 守卫,检查进行中两处按钮一并置灰,不会叠加请求。
- CLI:`wx-kit subscription check-now --accounts <fakeid,fakeid>`(fakeid 从 `subscription list` 取)。

## 改进

**`library list` / `search` 默认按发布时间降序(M31 · R3)**——为「每天拉取所有公众号最近文章清单」这个 agent 场景:

- 新增 `--sort publish|download|title` 与 `--order asc|desc`,**默认即 `--sort publish --order desc`**,agent 直接 `library list` 取前 N 条就是最近 N 篇。
- 空 publishTime 的条目在升降序下恒置末尾(与 GUI 同一份 `sortArticles`,逻辑抽到 core 共享,不双份维护)。
- ⚠️ **轻度 breaking**:默认输出顺序从「索引写入序」变为「publishTime 降序」。原顺序本就不确定、不应被依赖,新默认更可预期。

**`wx-kit -h` 给出仓库地址(M31 · R4)**——帮助末尾附 `https://github.com/monkeychen/wx-kit`,agent 遇到 skill 没覆盖的细节可自助去读 README / issues / releases。

## 修复

**mac 上 CLI 命令堆程序坞图标(M31 · R5)**——跑一次 CLI 冒一个图标、跑 N 次堆 N 个:

- 根因:Electron 在 mac 是 GUI 子系统应用,**进程一启动就在程序坞冒头,哪怕一个窗口都不开**;CLI 分支复用 Electron 进程(PDF 渲染需要 BrowserWindow)却没压制。
- 修复:CLI 分支在 `whenReady` 前 `app.dock.hide()`(仅 mac 存在,win/linux 天然 no-op)。GUI 启动的 dock 图标、CLI 的 PDF 离屏窗口均不受影响。
