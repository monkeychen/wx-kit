# wx-kit v0.7.0 — 磨平「下载 → 创作」链路的四处毛刺(2026-07-20)

本版不铺新平台、不改架构,只把「下载到的东西」和「用它创作」之间用户天天踩的四颗小石子磨掉。

## 改进

**Markdown 导出保留表格(M29 · R1)**——以前含表格的文章存成 md,表格直接散架变成一坨文本:

- 自写 GFM 表格规则转出标准 `| 列 | 列 |` 表;**不引 `turndown-plugin-gfm`**——插件产出的骨架对,但微信单元格外那层 `<section>` 会被当块级元素、炸出裸换行(非法 GFM),要修等于把插件核心重写,不如自己 40 行搞定,还少一个依赖。
- 单元格内行内格式(粗体/链接)、双引号、特殊符号原样保留;管道符转义。合并单元格(rowspan/colspan)按 GFM 限制尽力而为。
- 真实文章端到端验证:两张不同列数的表(2 列、3 列)均转成紧凑合法 GFM,GUI 阅读器渲染成真表格。

**应用内可见版本号(M29 · R2)**——以前只有 CLI `--version` 能看到版本号,GUI 完全不可见:

- 设置页新增「关于」区块:版本号(取自 `app.getVersion()`,发版自动跟上)+ 项目主页 / 查看新版本外链。
- (刊头版本号上线后即按反馈撤回——用户不关心版本号,常驻刊头是噪音;版本信息留「关于」即可查。)

**窗口标题栏去重(M29 · R3)**——原生标题栏那个「wx-kit」与应用内刊头品牌区信息重复:

- `BrowserWindow` title 与 `index.html` `<title>` 同改为空(mac 上标题栏只剩红绿灯 + 一条干净浅灰条),刊头零变化。一行配置改动、零风险。

**导出素材一键递给 agent(M30 · R4)**——以前导出素材后要「去 Finder 找路径 → 切到 agent → 自己拼提示词」三步:

- 导出成功由一条提示升级为 Modal:清单路径就地可见、可选中。
- 主按钮**「复制给 agent 的指令」**复制一段可直接粘进 Claude Code 等任意 agent 的自然语言(含清单绝对路径、`contentPath` 用法、引导「先读再确认选题」),省掉找路径和拼提示词两步。
- 调研后**否决**「直接唤起 Claude Code」:`claude` CLI 虽能带 prompt 起会话,但唤起的是新终端 + 陌生 cwd 的新会话,而用户真实场景是粘进已开着的会话——复制指令对任意 agent 通用、更贴合真实用法。

## 验证

- 单测 307 全绿(新增 GFM 表格 6 例 + agent 指令拼装 5 例)、tsc / lint / Playwright GUI e2e 全过。
- 真实文章端到端:下载含表格文章 → `content.md` 是合法紧凑 GFM 表格 → 阅读器 md 视图渲染成真表格。
- GUI 真机验证:导出 → Modal → 复制指令 → 主进程读回剪贴板逐字比对;清单 `contentPath` 逐条可达。
