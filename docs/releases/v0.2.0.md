# v0.2.0 · 2026-06-09

v0.1.0 之后的迭代:把下载闭环做扎实、信息架构理顺——「**下得放心、找得到、看得见**」。
五个里程碑 (M5–M9) 全部合入 main,端到端在真实微信公众号后台验证通过。

> **未签名**首次打开仍需手动放行,见 README「30 秒上手(下载安装包)」。macOS 签名+公证计划在后续版本。

---

## ✨ 新增功能

### M5 · 信息架构重构
- 导航三项:**下载 / 文库 / 设置**(去「批量」项,「书架」改名「文库」)。
- 「下载」页双模式:**按链接 / 按公众号**,编辑页签切换;默认进按链接,常驻开关可切。
- 双模式共享「保存为」配置卡,共用「开始下载」按钮。
- 字体规则:衬线=专名/标题,无衬线=正文/功能。

### M6 · 下载闭环 + 历史
- 下载完成后**就地确认**:每篇展示实际产出格式、可阅读、可在文件夹显示。
- **下载历史** `history.json` 索引(默认保留 1 年、可配置、支持清空):
  - 排序、分页懒加载(「加载更多」,非页码)。
  - **复制下载项**回填来源+范围+格式到配置卡再下一遍。
  - **每条记录可单删**;**失败行有重试**;**跳过项也保留**就地操作。
  - 清空/过期只清动作记录,绝不触碰已下文件与文库条目。

### M7 · 反馈引导
- **频控可见**:公众号列表阶段命中频控时,朱砂退避横幅 + 客户端倒数(「约 N 秒后重试 · 第 k 次」),不再像卡死;core 经 `onBackoff` 回调上报,IPC 透传为 `{kind:'backoff'}` 事件。
- **失败有指引**:把 `fetch timeout…` / `invalid or unavailable article` 等原始报错归一为「人话标题 + 下一步建议」,原始信息折叠在 tooltip 备查。
- 公众号**取消需二次确认**;**未下载的文章不蒸发**——补登记为 `cancelled` 进历史,以「未下载」徽章列出,提供单篇「下载」补下。

### M8 · PDF 保真
- 导出 PDF 不再跨页切图:在 `buildHtml` 的内联样式注入 `@media print { img/figure/table/pre/blockquote { break-inside: avoid } }`,只作用于打印态,屏幕阅读器零影响。
- 真实 PDF 出件 + 对照截图验证:无规则版切到第 17 行,有规则版整块下移完整。

### M9 · 文库组织
- **排序**:按下载时间 / 发布时间 / 标题,升降切换;空发布时间恒置末尾。
- **按公众号筛选** + **按公众号分组折叠**——两者可叠加。
- **批量选择 + 批量删除**(`library:removeMany`):只删记录不删文件,联动历史标记已删除。
- **卡片 ⇄ 列表** 视图切换(访达式):列表为紧凑行(缩略图·标题·公众号·发布·下载·行尾常驻操作);**列表分组时列头只显示一次**,各分区只留「印章+号名+篇数」分隔头;**单击=选中、双击=阅读**;底部浮动批量条选中即见、不挤动列表。
- 视图变换为纯逻辑 `src/renderer/library-view.ts`(TDD),只 import 类型——不破 renderer↔core 边界。

## 🔧 改进

- 测试: 119 单测 + 24 条 Playwright Electron e2e 全部绿;e2e 用真实缓存 session 跑公众号链路(无 session 退化登录门)。
- 错误处理: `explainError` 纯展示逻辑放 renderer 层(CLI 契约是 JSON,本不要人话),避免破「renderer 绝不 import core 运行时」的架构边界。

- README / 截图 / ROADMAP / PRD-v0.2.0 全部同步到 v0.2.0。
