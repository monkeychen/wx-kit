# wx-kit v0.5.4 — 订阅检查:不重跑、看得清失败、请求更省（2026-07-16）

## 修复与改进

**订阅检查加固（M22）**——源于安哥真机检查日志的三个问题:

- **同一时段重复检查**:检查耗时超过 1 分钟时,调度器下一个 tick 会再启动一个并发检查(真机曾同一时段两条相同记录,同批文章数了两遍)。现加防重入守卫:一次只跑一个;自动检查进行中手动点「检查更新」也只并入同一次。
- **「失败 x」可点开看明细**:检查记录里的失败此前只有个数,失败原因(频控/网络/登录失效)无从知晓。现在逐号失败原因(公众号名 + 错误)进检查记录,点「失败 x」弹窗查看;落盘日志与 CLI `subscription check-now` JSON 同步携带。
- **每号请求省 75%**:检查原固定取「最近 20 篇」——微信每页实回约 5 篇,要翻 4 页。现改「翻到水位为止」:日常每号 1 次请求即止;多日未开应用后自动翻深(封顶 20 篇),不漏文章。

**存储兼容**:检查记录新增可选字段 `failures`,旧记录不受影响(界面上保持纯文本)。

## 验证

- 单测 272 全绿(新增:防重入 tick 矩阵、失败明细三处落点、翻页到水位的请求次数断言)、tsc / lint / Playwright GUI e2e 全过。
- GUI 真机(seed 数据态):失败明细弹窗逐号显示「猫笔刀 — 微信频率限制（200013）」等;旧格式记录保持纯文本。
