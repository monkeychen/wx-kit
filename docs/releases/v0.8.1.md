# wx-kit v0.8.1 — 补上 v0.8.0 没真正修好的 dock 图标(2026-07-22)

v0.8.0 宣称修复了「mac 上跑 CLI 会在程序坞堆图标」,**实际没修好**。本版换了修法,真正解决。

## 修复

**mac CLI 不再冒程序坞图标(R1)**

- **v0.8.0 的修法为什么无效**:`app.dock.hide()` 放在 `app.whenReady()` 之前**并不生效**——dock API 要等 app ready 才被应用,而 AppKit 在那之前就已把进程注册成前台应用并画出了图标。实测 `wx-kit -h` 期间的进程状态序列是 `NULL → Foreground → UIElement`:图标先出现,随后才被隐藏。命令跑得越短,图标可见占比越大,`-h` 几乎全程可见。
- **本版修法**:打包配置加 `LSUIElement: true`(mac Info.plist),由 Launch Services 在进程启动时就定为 accessory,**不给图标出现的机会**——图标是 JS 执行前画的,只能在 plist 层拦。GUI 分支在 ready 后 `app.dock.show()` 把图标要回来,并 `app.focus({ steal: true })` 确保窗口正常到前台。
- 实测(打包态):`-h` 全程 40/40 采样为 `NULL`,从未出现 `Foreground`;`download` 为 `NULL → UIElement`;GUI 启动 `Foreground` + 窗口到前台,一切照旧。
- win/linux 不受影响(`LSUIElement` 是 mac 专有键)。
