# v0.3.0 · 2026-06-16

「列表顺手 + 公众号订阅」迭代。把文库列表视图做顺手，并新增类 RSS 的公众号订阅——让按公众号下载从「每次手动触发」升级为「订阅后周期检查、按需提示或自动下载」。

> **未签名**首次打开仍需手动放行，见 README「30 秒上手(下载安装包)」。

---

## 🗂 文库列表视图优化（M10）

- **列宽可调**：列表视图列头之间可拖拽调整列宽，松手后持久化进设置（重开 app 保持）。「发布时间」默认加宽到能完整显示分钟级时间。
- **表头排序**：点「标题 / 发布时间 / 下载时间」列头即按该列排序，列头显 ↑/↓ 指示，同列再点翻转方向。卡片视图保留工具栏排序入口。

## 📰 公众号订阅（M11）

新增「订阅」导航页（在「下载」与「文库」之间）：

- **订阅管理**：列出有 `fakeid` 的公众号（按公众号抓取历史 ∪ 搜号添加，去重），每个号可订阅/取消订阅；上方可搜号直接添加新订阅。
- **定时检查**：开启自动检查后，app 运行期到点检查已订阅号有没有发新文章（启动时补检当天一次）。
- **新文章处理**（设置可选）：仅提示（订阅页角标 + 逐号「下载 / 忽略」）或自动下载入库。
- 「新」判定用水位线（watermark）：订阅那一刻水位设为该号当前最新文章时间，只有订阅后发布的才算新，不会把存量一股脑当新文章。
- **诚实约束**：检查只在 app 开着时进行，无真后台；登录 session 过期不静默失败，订阅页显示登录引导。

## ⏰ 订阅触发升级 + 检查可观测性（M12）

- **两种触发模式**（设置可选）：每天某时刻 / 每隔 N 小时（网格锚定每天 0 点，N=6 → 0/6/12/18 点）。
- **检查可观测性**：订阅页「检查记录」倒序列最近 10 次（时间 · 自动/手动 · 查 N 号 · 新 M 篇 · 失败 K）；落盘日志 `subscriptions-check.log`（全量追加，可「打开日志文件」查看）；「下次预计检查」时间。**开了自动检查、到点后检查记录会冒出「自动」条目——这是定时生效的直接证据。**

## 🎨 UI 一致性打磨

- 四个导航页（下载/订阅/文库/设置）统一成满宽内容栏，切页面不再跳宽窄。
