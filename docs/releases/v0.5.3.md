# wx-kit v0.5.3 — 修复 macOS 关窗后程序坞无法重开窗口（2026-07-13）

## 修复

**macOS 程序坞激活重建窗口（M21）**——修复关闭窗口后点程序坞图标无响应、应用"假死"只能强退重开的问题：

- 根因：主进程只做了 mac 惯例的一半——关窗后应用驻留程序坞（`window-all-closed` 在 darwin 不退出），但**没有注册 `activate` 事件 handler**，关窗后不存在任何重建窗口的代码路径。缺陷自 v0.1.0 起即存在，开发/测试均为整进程启停故一直未暴露。
- 修复：`app.on('activate')` 时若无窗口则重建主窗口——点程序坞图标、`open -a` 已运行实例（同一 reopen Apple Event）均恢复正常；窗口存在时不会重复开窗。

## 验证

- 单测 260 全绿、tsc / lint 过；Playwright GUI e2e 新增「关窗 → activate → 界面完整重建 → 不重复开窗」断言全过。
- 打包态 .app 真机：关窗后 `open -a`（与点程序坞图标相同的 reopen 事件）窗口重建、界面完整，stderr 无异常。
